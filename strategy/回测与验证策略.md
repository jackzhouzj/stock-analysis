# 回测与验证策略（实盘前验证 · 把经验值变成数据值）

> 生成日期：2026-07-02
> 本模块解决体系的核心短板：**所有阈值都是"经验默认值"，未经验证就用真金实测**。
> 与《复盘与进化策略》配对——**复盘是"实盘后"验证，本模块是"实盘前"验证**，两者共同把校准结果收敛到 `阈值与版本记录.md`。
> 数据取数**优先走 iFinD MCP**（`get_stock_performance` 历史行情 / `index_data` 指数历史）。`search_stocks` 仅用于候选发现或抽样核验，不作为历史全市场母样本。
>
> **⚠️ 重要声明：本模块为验证方法论框架，不构成任何投资建议。回测/历史统计存在幸存者偏差、过拟合等固有局限，"回测通过 ≠ 实盘赚钱"。请独立决策、盈亏自负。股市有风险，投资需谨慎。**

## 〇、核心认知：回测不是万能，先分清能不能测

> 硬给"测不了的阈值"做回测，会因为还原不了当时的真实情况而**自欺**。第一步永远是分类。

### 阈值可回测性分类

| 类型 | 代表阈值 | 验证方式 |
|---|---|---|
| **可历史验证**（规则明确 + 有历史数据） | 止损幅度、盈亏比门槛、距启动涨幅 <30%/<50%、均线/放量形态、连板/涨停条件、估值分位 | 第 1 层：历史样本统计 |
| **难回测**（依赖当时主观/情绪/快照，历史难还原） | 双源共振主线、炸板率（历史全市场 breadth 难还原）、消息面预期差、外围09:31/14:31跨市场快照、超短线分时确认 | 跳过伪历史回测，直接第 2/3 层前向验证 |
| **条件可回测**（需自建历史分时库） | 超短线开盘区间、分时均价、尾盘强收盘、外围历史同刻覆盖、历史相同时刻量比 | 保存各市场逐分钟原始行情、时区/交易日历、实际涨跌停制度和当时板块快照后，执行严格point-in-time回测 |

> 判定原则：一个阈值若需要"当时才拿得到、事后无法准确还原"的数据，就不做历史回测，只做前向验证，避免前视偏差。

## 一、三层验证阶梯（从便宜到贵，逐层逼近真实）

```
第1层：历史样本统计   （成本最低·实盘前）→ 把"拍脑袋值"变成"历史样本支撑的初始值"
   ↓ 通过
第2层：模拟盘前向验证 （避免过拟合·实盘前）→ 抓历史测不到的滑点/执行偏差/情绪维度
   ↓ 通过
第3层：小仓位实盘校准 （最小真金·实盘中）→ 喂给《复盘与进化策略》→《阈值与版本记录》
```

### 第 1 层：point-in-time 历史事件回测

> 只有在可取得**当日可见的完整股票池与字段发布时间**时才允许执行。`search_stocks` 可用于候选发现和抽样核验，但不能证明某历史日全市场样本完整，禁止直接把其结果当成严谨回测母样本。

**数据契约（缺一不可）**
1. 每个交易日的 point-in-time 股票池，包含当时已上市、已退市、ST/*ST、退市整理、停牌和新股状态。
2. 财报、公告、股东和板块成分按首次披露/生效时间入库，禁止用当前值回填历史。
3. 信号价使用前复权序列保持技术指标连续；成交撮合使用当时可交易的不复权原始价格，并通过分红送转事件链计算总回报。
4. 每条数据保存来源、时间戳、单位、复权方式和版本；无法证明 as-of 时点的数据不得进入信号。

**事件驱动执行流程**
```text
1. 在决策时点仅用当时已知数据生成信号，冻结规则版本。
2. 下一可交易时点提交订单；按T+1、停牌、涨跌停、委托价格、成交量参与率和滑点判断是否成交。
3. 一字涨停买不到、跌停卖不掉、部分成交和撤单均保留在样本中，不得删除。
4. 按策略规则逐日模拟止损、止盈、时间退出和组合熔断，不得用“未来5/10/20日涨跌幅”替代真实退出路径。
5. 计入佣金、印花税、过户费、冲击成本、分红送转和现金变化后，计算净值、回撤与成交率。
6. 输出按环境和模式分层的成交率、期望、Profit Factor、最大回撤、最大连亏及置信区间。
```

**降级规则**：若只能用 `search_stocks` 抽取历史候选、无法证明股票池完整，则结果只能命名为“历史案例/条件样本统计”，不得升级阈值为“已完成历史回测验证”，也不得据此声称策略有效。

### 第 2 层：模拟盘 / 纸面前向验证

> 投真金前，按严格筛选策略**机械执行 1-3 个月**（模拟盘或纸面），积累前向样本。

- 目的：抓历史回测抓不到的问题——滑点、成交假设、执行时的情绪偏差、以及难回测的情绪/消息维度。
- 超短线双时点：分别在09:31—09:40与14:31—14:50保存候选池、实际快照、分钟Bar、板块快照、订单与T+1退出路径；M/A模式分开统计，禁止收盘后补写盘中信号。
- 外围覆盖：同步保存各市场当地交易日、时区、完成Bar、来源延迟、O等级、置信度、行业禁区、有效期和`overlay_id`；韩国或海外实时缺失必须原样留空。
- 外围验证：分别统计O3/O4后的A股次日/三日尾部损失、误报率、漏报率、折扣前后回撤和行业禁区效果，不能只挑“同跌”案例。
- 高频边界：未建立历史分钟快照库前，超短线只能标记“前向验证”，不能因日线条件统计通过而升级整套分时策略为🟡历史验证。
- 要求：**完全按纪律执行，不中途改规则**（否则验证的不是策略而是临场发挥）。
- 记录：用《复盘与进化策略》的统一交易日志，标注"模拟盘"。

### 第 3 层：小仓位实盘 + 复盘校准

> 最小真金验证，快速迭代。用《资金与风险管理策略》的最小仓位，走《复盘与进化策略》闭环。

## 二、五大防坑清单（回测最容易骗自己的地方）

| 坑 | 表现 | 防范 |
|---|---|---|
| **前视偏差** | 用了当时拿不到的数据（未来价格、事后复权、事后才知的主线） | 只用"截至样本日"能拿到的数据；复权口径固定，不用未来信息 |
| **过拟合** | 在同一段历史反复调参到"完美"，实盘失效 | 样本内调参、样本外验证分开；参数越少越稳健 |
| **幸存者偏差** | 只统计还在的票，漏掉退市/一字跌停出不掉的 | 样本含已退市/ST；把"卖不掉"计为最坏结果 |
| **样本量不足** | 10 笔就下"胜率 60%"结论 | 至少几十~上百笔，分环境各自够量才有效 |
| **不计成本** | 忽略印花税+佣金+滑点 | 每笔计入单边成本，往返约 0.1%-0.3% |

> 铁律补充：**分牛熊/分环境统计**——脱离环境（进攻期/退潮期）谈胜率没有意义（与《大盘环境模块》口径一致）。

## 三、验证结果如何回填《阈值与版本记录》

> 每个阈值标注**验证状态**，一眼看出哪些还是纯拍的、哪些已被数据验证。

| 验证状态 | 含义 |
|---|---|
| 🔵 经验默认 | 初始拍的值，未经任何验证 |
| 🟡 历史验证 | 经第 1 层历史样本统计支撑（标注样本量、区间、分环境结果） |
| 🟢 实盘校准 | 经第 2/3 层前向/实盘验证并回填（标注样本量、周期） |

```
回填流程：
历史样本统计 → 更新阈值验证状态为🟡 + 记录样本依据
模拟盘/实盘复盘 → 更新为🟢 + 记录实测胜率/盈亏比
所有状态变更写入《阈值与版本记录》第二节"调整历史"，禁止无痕改。
```

## 四、验证优先级（先验证影响最大的）

> 不必一次验证所有阈值，按"错了代价最大"排序：

1. **止损幅度 + 盈亏比门槛**——直接决定亏损大小，最该先验。
2. **超短线双时点执行**——先验证M/A成交率、滑点、T日失效率、标准化跳空/涨幅和1—3日时间退出；无历史分时库时只做前向验证。
3. **外围市场O3/O4覆盖**——验证标准化阈值、误报/漏报、风险预算折扣和恢复条件；无跨市场point-in-time数据时仅做前向验证。
4. **买点形态胜率**（放量突破/龙回头等）——决定进攻端有效性。
5. **距启动涨幅 / 位置阈值**——决定赔率。
6. **打分卡权重**——用实测数据回填哪个维度解释力最强。
7. 情绪/主线/消息类（难回测）——直接进模拟盘前向验证。

## 五、现实边界（诚实说明）

- 真正严格的量化回测需要**历史数据落库 + 事件驱动引擎**，是更大的工程（见 `tasklist.md`）。
- 本模块定位是**轻量验证**：历史样本统计 + 模拟盘前向验证，投入产出比最高，适合 AI 驱动的个人系统。
- **回测/历史统计通过，只代表"过去这样"，不保证未来**。它降低"用纯拍脑袋值实盘"的风险，但不消除市场不确定性。

---

## 一句话总览

> **先分清能不能测，再用"历史样本统计 → 模拟盘 → 小仓位实盘"三层逼近，全程防五大坑、分环境统计，把每个阈值从🔵经验默认逐步验证到🟢实盘校准。** 回测降低自欺，不承诺盈利。
