# 账户分析策略（自动读持仓 · 逐票诊断 · 组合风控 · 操作清单）

> 生成日期：2026-07-02
> 本策略自动读取 `account/{账户名}.csv` 中的持仓，用 iFinD MCP 补全实时数据，对**每只持仓逐票诊断**并做**组合层面风控**，最后输出**可执行的操作清单**（加仓/持有/减仓/清仓 + 价位 + 理由）。
> 复用模块：`个股综合体检策略.md`（逐票体检）、`大盘环境模块.md`（环境/熔断）、`资金与风险管理策略.md`（仓位/集中度/熔断口径）、`卖出决策策略.md`（止损止盈减仓）、`持仓管理.md`（台账字段）、`策略路由.md`（跨线集中度）。
> 取数**优先走 iFinD MCP**，MCP 不可用时才回退脚本。
>
> **⚠️ 重要声明：本策略不构成任何投资建议、不预测涨跌、不推荐个股。所有分析基于历史数据与规则框架，市场瞬息万变。请独立研究、独立决策、严控风险、盈亏自负。股市有风险，投资需谨慎。**

## 〇、使用说明

### 输入
- **账户名**（必填）：对应 `account/` 目录下的 CSV 文件名，如 `erik` → 读取 `account/erik.csv`。
- **持仓意图**（可选）：若某票已明确归属短/中/长线，按对应线纪律诊断；未标注则自动判定归属线。

### 触发
- "帮我分析账户 / 分析我的持仓 / erik 账户怎么操作 / 我的仓位该怎么调"

### 输出
- 账户总览（总市值、总浮盈亏、单票占比、行业集中度、环境分）
- 逐票诊断表（现价、浮盈亏、五维体检、归属线、操作建议）
- 组合风控提示（集中度、系统性熔断、总仓位）
- 操作清单（每票：加仓/持有/减仓/清仓 + 触发价位 + 理由）

## 一、数据读取与字段规范

### CSV 必填字段（现有）

| 列名 | 含义 | 示例 |
|------|------|------|
| 股票名称 | 证券简称 | 紫光国微 |
| 股票代码 | 6 位代码 | 002048 |
| 成本线 | 每股持仓成本 | 88.75 |
| 仓位（股） | 持有股数 | 400 |

### 可选字段（建议补充，缺失则自动处理/降级）

| 列名 | 含义 | 缺失时的处理 |
|------|------|--------------|
| 策略归属 | 短线/中线/长线 | 未标注 → 按第三节自动判定归属线 |
| 建仓日期 | 首次买入日 | 用于估算持有周期，缺失则跳过 |
| 止损位 | 已设定的止损价 | 缺失 → 按归属线纪律 + 波动率自适应计算 |
| 备注 | 买入逻辑 | 缺失则不校验"逻辑是否兑现" |

### 账户资金（用于仓位占比与固定风险法）
- 若 CSV 顶部或单独提供 `账户总额`（含现金）→ 用它作分母算仓位占比、算固定风险法。
- **若未提供** → 以「持仓市值合计」作分母算**相对持仓占比**，并**明确标注"现金未知、总仓位无法计算"**，不臆测账户总额。

## 二、数据补全（iFinD MCP 优先）

> 对 CSV 里每只票，按需调用以下工具补全实时数据。同一工具支持多主体，尽量合并查询（≤5 只/次）。

| 补全项 | 取数工具（MCP 优先） | 用途 |
|--------|----------------------|------|
| 现价、涨跌幅 | `stock_highfreq_quotes`（实时快照） | 算浮盈亏、当前位置 |
| 均线/趋势/距启动涨幅 | `get_stock_performance` | 技术面、位置 |
| 主力资金净流入 | `get_stock_performance`（口径=**净主动买入额**） | 资金面 |
| ROE/毛利/现金流/营收 | `get_stock_financials` | 基本面 |
| PE/PB 分位、PEG | `get_stock_financials` | 估值 |
| 减持/质押/违规/解禁 | `get_stock_events` / `get_stock_shareholders` / `search_notice` | 排雷 |
| 波动率/beta | `get_risk_indicators` | 波动率自适应止损 |
| 所属行业/板块 | `get_stock_info` / `sector_data` | 组合集中度 |
| 大盘环境 | 见 `大盘环境模块.md`（`index_data`/`index_highfreq_quotes`/plate-rotation） | 组合级熔断、择时 |

**衍生计算**
```
浮盈亏%   = (现价 - 成本线) ÷ 成本线
浮盈亏额  = (现价 - 成本线) × 仓位（股）
单票市值  = 现价 × 仓位（股）
持仓总市值 = Σ 单票市值
单票占比  = 单票市值 ÷ (账户总额 或 持仓总市值)
```

## 三、逐票诊断

> 每只持仓走一遍精简体检（复用 `个股综合体检策略.md` 的五维），并结合**持仓成本**给出操作建议。

### 3.1 归属线自动判定（无"策略归属"时）
| 归属线 | 判定线索 |
|--------|----------|
| 短线 | 持有 <2 周、买在题材主线、技术驱动为主 |
| 中线 | 持有数周~数月、趋势+景气驱动 |
| 长线 | 持有数月以上、基本面+估值驱动 |

> 判定后**按对应线的纪律诊断**（止损宽度、卖出条件、复检频率各不相同）。归属不清时默认按更严的短线纪律保护本金。

### 3.2 单票诊断项
| 维度 | 判定 | 取数 |
|------|------|------|
| 浮盈亏 | 盈利/亏损幅度，是否已触及止损位 | 现价 vs 成本线 |
| 排雷复检 | 是否新增减持/质押/违规/解禁/暴雷 | `get_stock_events`/`search_notice` |
| 基本面 | ROE/成长/现金流是否仍健康（中长线重点） | `get_stock_financials` |
| 技术面 | 趋势/均线/距启动涨幅/量价 | `get_stock_performance` |
| 资金面 | 主力近 5-10 日净流入/流出 | `get_stock_performance` |
| 估值 | PE/PB 分位、PEG（G 用可持续增速） | `get_stock_financials` |
| 板块状态 | 所属板块是否主线/退潮 | `plate-rotation` |

### 3.3 单票操作建议映射
```
IF 排雷复检命中新雷（减持加速/违规/暴雷/解禁洪峰）
THEN → 减仓/清仓（不看浮盈亏，治理雷优先）

IF 已触及止损位 OR 跌破归属线的止损逻辑
THEN → 止损清仓（涨跌停无法成交时按《制度性风险应对》处理）

IF 归属线逻辑仍成立（短线:形态+资金未退潮 / 中线:趋势+景气未破 / 长线:基本面未变）
THEN → 持有；若同时满足加仓条件（回踩不破/越跌逻辑未变且未达仓位上限/赔率仍≥门槛）→ 评估加仓

IF 触及止盈信号（短线:放天量滞涨/破均线 / 中线:破趋势/景气证伪 / 长线:估值极端）
THEN → 减仓/清仓
```

### 3.4 止损位计算（缺"止损位"字段时）
- 按归属线基准：短线 -5%~-8%、中线 -8%~-10%、长线看基本面不设价格止损。
- **波动率自适应**（`get_risk_indicators` 取波动率/beta）：高波动票放宽、低波动票收紧。
- 已盈利的票用**移动止盈**替代固定止损（如跌破 5/10/20 日线分级减仓）。

## 四、组合层面分析

### 4.1 组合总览
- 总市值、总浮盈亏（额+%）、盈利/亏损票数、最大单票占比。
- 若有账户总额：总仓位%、现金比例。

### 4.2 集中度风控（复用《策略路由》跨线集中度约束）
- **单票占比**：警戒线（如单票 >25% 持仓市值）→ 提示过度集中。
- **单一行业/产业链敞口 ≤35%**：用 `get_stock_info` 行业 + `sector_data` 板块归类，识别"假分散"（多票押同一链）。
- 超限 → 提示优先减持集中赛道，而非继续加仓。

### 4.3 系统性熔断（复用《大盘环境模块》）
```
先按《大盘环境模块》算环境分：
  环境分 <30（大盘破位/无主线）→ 提示整体降仓（短线优先清、中线降至底仓、长线暂停加仓保现金）
  环境分 30-49（退潮/谨慎）    → 提示降低总仓位、只保留最强逻辑的持仓
  环境分 ≥50                  → 按逐票诊断正常执行
```

## 六、执行纪律与口径（硬约束）

1. **所有诊断必须基于真实数据调用**（MCP 优先），禁止凭印象编造现价/财务数字；取不到就如实说明。
2. **归属线不混用**：短线票破位就按短线止损，不改口"做长线"扛套。
3. **治理雷优先于浮盈亏**：命中新雷先减/清，不因"还在赚"而留。
4. **止损考虑 A股制度**：涨跌停/一字板无法成交时按《短线严格筛选策略》制度性条款处理；T+1 有一天空窗。
5. **总仓位无法计算时明确声明**：账户总额缺失就只给相对持仓占比，不臆测。
6. **不预测涨跌、不承诺收益**，操作建议是规则推演，决策权在用户。

## 七、多账户支持

- `account/` 下每个 `{名}.csv` 是一个独立账户，调用时指定账户名即可。
- 不同账户可有不同风格（如 `erik.csv` 偏成长、`xxx.csv` 偏价值），诊断时按各账户实际持仓独立分析。
