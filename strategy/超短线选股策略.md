# 超短线选股策略（双时点确认 · 1—3日持有 · T+1约束）

> 生成日期：2026-07-21
> 本策略用于 A 股超短线候选筛选与执行，设置“开盘确认窗”和“尾盘确认窗”两种互斥模式。
> 它不是预测次日涨停的模型，而是在环境、主线、资金、量价、位置和可成交性均通过后，以小风险预算参与短周期惯性。
> 环境、外围价格覆盖、账户风险和组合熔断引用 `strategy/大盘环境模块.md`、`strategy/外围市场影响模块.md` 与 `strategy/资金与风险管理策略.md`；
> 数值唯一事实来源为 `strategy/阈值与版本记录.md`；卖出统一引用 `strategy/卖出决策策略.md`。
>
> **重要声明：超短线受 T+1、隔夜跳空、涨跌停和高摩擦成本影响，计划止损不等于实际最大亏损。本文仅用于策略研究，不构成投资建议。**

---

## 〇、策略边界

| 项目 | 定义 |
|---|---|
| 市场 | A 股普通股票；不含 ST/*ST、退市整理、上市未满20个交易日和无涨跌幅限制期标的 |
| 持有周期 | 计划持有 1—3 个交易日；到期不转中线，不因亏损改口“长线” |
| 开盘确认窗 M | 09:31—09:40；09:30整不使用未完成1分钟Bar直接下结论 |
| 尾盘确认窗 A | 14:31—14:50；14:30整先等当前1分钟Bar完成，不在14:57后追单 |
| 信号模式 | 每笔交易只能标记 M 或 A；禁止事后切换模式美化结果 |
| 方向 | 只做多，不抄明确下降趋势，不参与一字板和预计无法成交标的 |
| 核心假设 | 主线持续 + 个股相对强 + 当日结构确认，争取次日或三日内延续 |

### 为什么不在 09:30:00 直接买

- 09:30附近价差、冲击成本和假动作最集中，首个1分钟Bar尚未完成。
- 当前高频工具只提供当日日内数据，不能自动取得历史同刻分时作严格比较。
- 因此先在 T-1 收盘后建立候选池，开盘只确认，不临时全市场追涨。
- 若必须更早执行，只能另接可靠集合竞价/L2数据，并单独验证；当前策略不假装拥有这些数据。

## 一、总流程

```text
T-1收盘后建立候选池（最多10只）
  → 决策日依次过A股环境、外围价格、消息事件、板块和全局排雷
  → 选择 M 或 A 单一模式做分时确认
  → 硬门槛通过后评分与赔率校验
  → 按压力风险法和超短线上限确定仓位
  → T+1起按逻辑止损、R倍数保护和时间退出执行
  → 每笔写入统一交易日志并前向验证
```
## 二、共享硬闸（任一不过即不交易）

### 闸门 0：环境与账户状态

| 条件 | 动作 |
|---|---|
| 环境分≥75，赚钱效应和主线清晰度均非0，外围非O4 | M/A模式正常开放 |
| 外围O3且置信度A/B | 仅非禁区候选，单笔风险预算乘台账折扣；与防守环境规则取更严 |
| 外围O4且置信度A/B | M/A模式冻结新仓；存量按卖出和T+1规则处理 |
| 外围O0或C级 | 不得视为中性；核心市场缺失时只输出观察名单 |
| 环境分50—74 | 仅最强双源共振主线的一档核心；使用防守仓位上限 |
| 环境分<50 | 超短线不开新仓；环境分<30时服从全局风险冻结 |
| 账户/组合熔断、未成交止损单或开放风险超限 | 禁止新增超短线风险 |
| 环境关键数据缺失 | 只输出观察名单，不生成可执行信号 |

消息面按 `strategy/消息面事件驱动模块.md` 检索截至 `decision_time` 的最近3个自然日：

- 高置信度L1/L2系统性利空未被外围价格层覆盖：关闭M/A模式或设置行业禁区；已有同一`overlay_id`时只合并证据，不重复降级。
- 超预期利好只提高方向优先级，不放松硬门槛；已连续上涨或利好不涨时反而降级。
- 新闻无结果不等于无风险；具体公司必须用公告/结构化事件交叉验证。

### 闸门 1：板块与个股身份

1. 板块必须来自 plate-rotation 双源共振Top10，或T-1为持续主线且决策时点仍有盘面确认。
2. 个股必须是板块确认龙头、准龙头或最强补涨，不能只因“还没涨”入选。
3. 龙头已断板、板块排名连续下滑且无新龙接力时，整个板块退出超短线候选池。
4. kaipan强度分与ths涨幅只能各自排序，严禁跨源比较或称作资金净流入。
5. 盘中调用板块工具要禁用过期缓存或使用≤60秒TTL，并记录真实快照时间。

### 闸门 2：通用排雷与可成交性

以下任一命中即淘汰：

- ST/*ST、退市整理、正式退市风险警示、上市未满20个交易日或无涨跌幅限制期。
- 立案调查、财务造假认定、重大处罚、重大诉讼/减持/解禁尚未充分评估。
- 一字涨停、停牌、预计无法以合法价格成交，或盘口/流动性不足以控制冲击成本。
- 近20日日均成交额低于阈值，或计划订单超过超短线成交参与率上限。
- 近5日量增价跌、放量长上影、资金持续派发，或处于60日线明确向下的下降趋势。
- 硬风险字段查询失败、证券主体映射不唯一或数据时间晚于决策时点。

> 排雷细节复用《短线严格筛选策略》的R-1～R-7；后续建立统一排雷模块时改为直接引用，本文不另造较宽标准。

### 闸门 3：T-1候选池

T-1收盘后筛选，不在开盘瞬间临时扫描全市场：

| 维度 | 候选要求 |
|---|---|
| 环境 | 下一交易日未处于已知全局风险冻结；重大隔夜事件次日重新评估 |
| 外围覆盖 | 保存最近完成的美股、港股和可得韩国市场状态、O等级、置信度与有效期；缺失不填中性 |
| 板块 | 双源共振、排名改善或稳定，非连续3日退潮 |
| 趋势 | 收盘站上20日线，20日线走平或向上；5/10日结构不空头发散 |
| 相对强度 | 最近1—3日强于板块/指数，回调时缩量，不靠单日脉冲 |
| 资金背景 | 按《主力资金流分析模块》取近5日+近1—3个已完成交易日，非持续派发 |
| 位置 | 非≥5板/连续一字/严重乖离，不在明显加速末端 |
| 赔率 | 到客观压力位的预估空间至少为压力退出空间的2倍 |
| 数量 | 最终候选≤10只，以符合高频接口单次主体上限 |

候选池必须保存：证券代码、所属板块、候选原因、T-1收盘价、压力退出位、目标位、实际涨跌停宽度、日均成交额和硬风险查询状态。

## 三、资金与量价口径

- 历史资金只使用 `get_stock_performance` 的主力/特大单**净主动买入额**，并明确起止日期、单位和方向。
- 超短线窗口为近5个已完成交易日看背景，近1—3个已完成交易日看边际。
- 决策日的 `stock_highfreq_quotes` 只确认价格、均价、成交量和换手，不能把分时OHLCV冒充“主力净流入”。
- 近5日背景持续净流出且近1—3日仍放量流出：硬否决。
- 近5日净流入、近1—3日延续净流入且量价健康：优先。
- 单日巨额脉冲后次日反向：淘汰或降入观察池。
- 无本地历史同刻分时库时，不计算伪精确“相同时刻量比”；该项标记缺失，不用历史全天量替代。

## 四、模式 M：开盘确认（09:31—09:40）

### 4.1 决策时钟

```text
09:25后：只在有可靠竞价源时记录竞价事实；当前工具无竞价/L2能力，不据此打分
09:30—09:31：观察，不使用未完成1分钟Bar下单；同步按外围模块读取各市场已完成数据
09:31起：可读取A股首个完整1分钟Bar；港股也只能采用供应商确认完成的Bar
09:35左右：优先用首个完整5分钟开盘区间确认
09:40后：M模式停止开新信号；错过不追，等待A模式重新独立评估
```

### 4.2 标准化跳空

不同板块涨跌停宽度不同，统一归一化：

```text
标准化跳空 = (开盘价 / T-1收盘价 - 1) ÷ 当日实际涨停宽度
```

- 落在阈值台账区间内才继续；负跳空必须在确认窗内重新站回T-1收盘或分时均价。
- 超过上限视为高开透支，不追；一字或接近涨停且无法正常成交直接淘汰。
- 实际涨停宽度必须按普通主板、创业板/科创板、北交所等当日制度获取，不固定用9.9%。

### 4.3 两种入场结构（二选一）

**M1 开盘区间突破**

1. 首个完整5分钟区间未出现高开低走，低点未被放量击穿。
2. 价格位于开盘价和当日均价之上，板块与指数没有同步转弱。
3. 09:35后放量突破开盘区间高点，突破Bar收在区间上沿之上。
4. 若没有历史同刻量能，只能写“当日攻击量持续”，不能声称量比达到历史阈值。

**M2 首次回踩承接**

1. 开盘冲高后回踩当日均价/开盘价，回踩缩量且不破首个完整分钟低点。
2. 随后重新站上当日均价并突破回踩前小高点。
3. 板块核心同步走强，个股不是孤立脉冲。

### 4.4 M模式一票否决

- 09:30后急拉但价格持续跌破当日均价，高点和低点同步下移。
- 首5分钟出现放量长上影、量增价跌或标准化跳空超过上限。
- 个股上涨但所属板块排名/核心股同步转弱。
- 首次突破失败后短时间连续追单；M模式每只股票当天只允许一个计划买入批次。
- 关键分时、环境或板块数据在确认窗内未取得；宁可错过，不补写信号。

## 五、模式 A：尾盘确认（14:31—14:50）

### 5.1 决策时钟

```text
14:30：记录A股快照，等待当前1分钟Bar完成；外围市场分别使用截至该时点已完成数据
14:31—14:40：核验全天结构、板块、均价、日内高低点、外围覆盖和成交状态
14:40—14:50：仅在触发结构完成且风险预算充足时执行
14:50后：不新开A模式订单；14:57后的收盘集合竞价不作为追单窗口
```

### 5.2 标准化日涨幅

```text
标准化日涨幅 = (决策价 / T-1收盘价 - 1) ÷ 当日实际涨停宽度
距日内高点 = (日内最高价 - 决策价) ÷ 日内最高价
```

必须同时满足：

- 标准化日涨幅位于台账区间，不做弱势横盘，也不追临近涨停的加速末端。
- 决策价在开盘价和当日均价之上，且距日内高点不超过台账阈值。
- 13:00后低点不持续下移，尾盘不是急拉诱多或放量滞涨。
- 当前累计成交额已达到基本可成交要求；历史同刻量比仅在本地有可审计快照时使用。

### 5.3 三种尾盘结构（至少一种）

**A1 午后平台突破**：13:00后窄幅整理，14:31后放量突破平台上沿，突破后未快速跌回。

**A2 均价承接强收盘**：全天多数时间在当日均价之上，午后回踩缩量，尾盘重新走强并接近日内高点。

**A3 板块二次回流**：板块午后排名/强度重新改善，核心股同步回流，候选股相对板块更强且非孤立拉升。

### 5.4 A模式一票否决

- 决策价位于当日均价下方，或14:30后才突然无持续性的直线拉升。
- 高位巨量长上影、日内高点连续下移、尾盘放量却不涨。
- 板块龙头炸板/断板且无新龙接力，或双源主线明显退潮。
- 预计次日仅有不足2R的空间，或隔夜事件风险无法评估。
- 早盘已买入同一标的后，尾盘以“确认”为名加仓；T日不可卖风险不得扩大。

## 六、硬闸通过后的评分卡

> 打分只排序，不能补救任何硬门槛。M与A分别统计，禁止混在一起计算胜率。

| 维度 | 满分 | 评分规则 |
|---|---:|---|
| 环境与板块 | 25 | 环境≥75、双源共振、主线持续、决策时点未退潮 |
| 日频资金与相对强度 | 15 | 近5日+近1—3日非脉冲净流入，个股强于板块/指数 |
| 分时入场结构 | 25 | M1/M2或A1/A2/A3结构完整；价格在均价上、无出货量型 |
| 位置与赔率 | 15 | 不在加速末端，压力退出清晰，计成本后预估盈亏比≥2:1 |
| 流动性与执行 | 10 | 日均成交额、参与率、合法委托、非一字/停牌均通过 |
| 排雷与数据质量 | 10 | 主体、时点、单位、风险字段完整；无未来数据和关键缺失 |
| **合计** | **100** | 一档/二档线以阈值台账为准 |

评分处理：

- 硬门槛字段缺失：进入观察池，不评分入选。
- 非关键加分项缺失：得0或标记不可评估，不默认给满分。
- 总分达标仍必须通过风险预算、集中度、组合开放风险和可成交性检查。

## 七、仓位与订单纪律

### 7.1 压力风险法

```text
基础风险幅度 = max(结构止损幅度, 1.5×ATR幅度, 历史隔夜不利跳空分位数)
压力风险幅度 = 基础风险幅度 + 滑点/冲击成本 + 涨跌停或停牌附加
风险定仓金额 = 账户净资产 × 超短线单笔风险容忍度 ÷ 压力风险幅度
实际仓位 = min(风险定仓金额, 超短线单票上限, 流动性上限, 组合剩余风险对应仓位)
```

- 结构止损优先使用M模式开盘区间低点/均价失守，或A模式午后平台/均价失守。
- 止损幅度硬上限不等于实际最大亏损；T+1跳空和跌停可能显著穿透。
- ATR、跳空或流动性数据缺失时降仓或不交易，不假定能在止损价成交。

### 7.2 订单约束

1. 一档、二档、超短线组合、同板块和防守环境上限均以阈值台账为准。
2. 每只股票每个交易日只允许一个计划买入批次；不追单、不补仓、不摊低成本。
3. 每日新增标的数和计划风险合计不得超过台账上限；同板块多个候选只保留最高分者优先。
4. 委托必须记录计划价、合法价格边界、数量、已成交/未成交数量、滑点和撤单原因。
5. 部分成交按真实数量建仓；未成交不得在复盘中记为“策略盈利”。
6. M和A模式共享同一超短线风险池，尾盘不得绕过早盘已占用的风险预算。

## 八、T+1卖出与1—3日时间退出

### 8.1 T日不可卖风险

当日买入后，即使结构立即失效也不能卖出：

- 立即取消该票所有未成交买单，禁止加仓。
- 把当前持仓记为不可卖lot，继续占用组合开放风险。
- 记录T日失效原因，并在下一可卖日优先处理。
- 回测和复盘不得虚构T日止损成交。

### 8.2 下一交易日处理顺序

```text
1. 有无重大利空、板块退潮、跌停或无法成交？
   → 有：按合法价格优先降低风险，未成交继续计提压力风险。
2. 开盘后是否重新站上T-1关键价/当日均价，板块是否仍强？
   → 否：退出或大幅减仓，不等“解套”。
3. 是否达到+1R/+2R且趋势仍在？
   → 按盈利保护与分批止盈执行。
4. 到时间退出点仍无延续？
   → 清仓，不转普通短线或中线。
```

- 对可卖仓位，跳空低开且首个完整5分钟无法收复T-1收盘/当日均价，按逻辑失效处理。
- 一字跌停、停牌或委托未成交时，挂单不等于卖出；下一时点继续按真实状态处理。
- 盘中单次波动不能被事后解释成“洗盘”以取消原止损。

### 8.3 R倍数盈利保护

```text
R = 入场价 - 压力退出价（按每股计算）
达到+1R：计划保护线提高到成本附近，考虑费用和正常滑点
达到+2R：至少减仓1/2；剩余仓位跟踪分时均价、前一日低点或5日线
趋势破坏：清仓剩余，不等待固定目标价
```

保护线是计划委托依据，不承诺跳空后能在该价成交。

### 8.4 时间退出

- T+1收盘前仍未表现出相对强势，且板块/资金边际转弱：退出。
- 最迟T+2仍未创新高或触发至少+1R：退出，不继续占用超短线额度。
- 无论盈亏，T+3收盘前原则上清仓；只有已达到+2R且剩余仓位按移动保护执行时，可保留极小利润仓，但必须转为新的普通短线记录，不能篡改原策略统计。
---

## 九、专属熔断与禁做清单

### 9.1 专属熔断

- 连续亏损或近5日超短线累计净亏损达到台账阈值：冻结超短线新开仓1个交易日并复盘。
- 当日超短线计划风险已达到每日上限：M/A模式均停止新增。
- 决策时点环境跌破50、赚钱效应与主线清晰度同时为0：停止新增超短线仓位。
- 账户单日回撤、月度回撤或全局冻结先触发时，按《资金与风险管理策略》执行，不得以专属阈值更宽为由继续交易。

### 9.2 永久禁做行为

- 09:30整无确认抢单，或14:50后因害怕错过而追单。
- 盘中临时扩大候选池，用事后涨幅反选“当时本可买到”的股票。
- 把板块强度、涨幅、成交额或分时放量称作主力净流入。
- 当日买入后补仓摊低成本，或亏损后改成普通短线/中长线。
- 忽略T+1、一字板、未成交和部分成交，按理论止损价美化结果。
- M与A模式混算、盘中改阈值、只记录成功信号而删除失败/未成交样本。

### 高频硬边界

1. 高频接口仅支持交易日日内，不能查询过去交易日09:31或14:31的分钟序列。
2. 没有本地历史分时库前，M/A模式只能做纸面/模拟盘前向验证，不能宣称完成严格历史回测。
3. `search_stocks` 不保证全市场point-in-time穷尽，不能用它证明历史候选池完整或计算全量炸板率。
4. 当前工具未明确提供集合竞价、封单队列、炸板次数和回封时间；相关项不进入硬门槛与评分。
5. 自然语言返回必须校验证券代码、市场、指标、单位、日期、时间戳和复权方式。

## 十三、复盘与验证

每笔至少记录：

- 模式M/A、实际决策时间、数据截至时点和规则版本。
- 候选池是否在T-1生成，是否存在临时追涨。
- 环境、板块、资金、分时结构、评分、计划R与实际仓位。
- 理论成交与真实成交差异、滑点、费用、部分成交和未成交。
- T日是否立即失效、T+1可卖状态、实际退出路径和持有交易日数。
- MFE、MAE、是否达到+1R/+2R、退出原因和纪律偏差。

验证要求：

1. M与A分开统计成交率、胜率、期望、Profit Factor、最大回撤和最大连亏。
2. 在没有历史分钟快照前，至少连续前向记录1—3个月，不回填遗漏信号。
3. 首轮优先验证止损/赔率、标准化跳空/日涨幅、决策窗口、分时结构和流动性阈值。
4. 每个模式至少积累足够样本后再调整；所有调整先写入《阈值与版本记录》并留痕。
5. 规则未通过验证前均保持🔵经验默认，不对外声称胜率或收益率。
