# 消息面事件驱动模块（共享·事件解释与二次修正）

> 生成日期：2026-07-02
> 最近修订：2026-07-21（与外围市场价格覆盖层分工并防重复计权）
> 本模块是**全策略共用的消息面/事件判断口径**。它在 `strategy/外围市场影响模块.md` 之后执行，负责解释价格冲击、验证预期差并对 A 股初始路由做一次最终定性修正。
> 数据取数**优先走 iFinD MCP 工具**（`search_news` / `search_notice` / `get_edb_data`），MCP 不可用时才回退脚本。
>
> **⚠️ 重要声明：本模块为消息面判断辅助框架，不构成任何投资建议。消息面驱动存在时效性与证伪风险，所有信号为事后确认而非预测。请独立决策、盈亏自负。股市有风险，投资需谨慎。**

---

## 〇、定位：修正项，不是买入触发器

> **消息面只修正"环境分级 + 仓位 + 方向"，绝不作为唯一买入理由。** 买入触发永远回到个股的「形态 + 资金 + 位置」三共振。

- 本模块**不机械加进大盘环境模块的 0-100 分**。原因：消息是定性的，机械打分会失真；更重要的是，机械给消息加分会直接诱发《短线炒股策略》第九条明令避开的**"追消息"坑**。
- 它的作用有三种：**降级否决**（利空）、**方向加权**（超预期利好，仅作主线参考）、**见顶警示**（利好出尽）。
- 外围实际涨跌、Bar 完成状态和 O0—O4 等级统一引用《外围市场影响模块》；本模块不再凭新闻标题推断市场涨跌。
- 同一冲击必须关联 `overlay_id`。若外围价格层已经执行风险动作，新闻层只能补充原因、置信度、行业映射或有效期，不得再次降级。

## 一、消息分级（影响力从大到小）

| 级别 | 类型 | 典型事件 | 对 A 股的作用范围 |
|---|---|---|---|
| **L1 宏观流动性** | 货币政策 | 美联储加息/降息、非农、国内降准降息、MLF/LPR | 系统性，影响全市场风险偏好与估值锚 |
| **L2 产业政策** | 国家/行业政策 | 以旧换新、AI+、低空经济、设备更新、行业整顿 | 板块级，直接催化对应赛道 |
| **L3 行业景气** | 供需/价格/订单 | 涨价函、龙头订单、产能出清、海外映射 | 板块级，趋势性但发酵较慢 |
| **L4 个股事件** | 公司层面 | 业绩预告、并购重组、减持、中标、诉讼 | 个股级，注意真伪与预期差 |

> 级别越高，对**环境分级**的修正权重越大；级别越低，越偏个股层面（归入排雷/体检环节处理）。

## 二、事件 → 板块映射表（速查）

### 外因（海外传导到 A 股）

| 消息 | 传导路径 | A 股利好方向 | A 股利空方向 |
|---|---|---|---|
| 美联储**加息** / 非农超预期强 | 美元↑ → 人民币承压 → 北向流出 → 风险偏好↓ | 有限（避险有时反被强美元压制） | 成长股、科技、高估值赛道 |
| 美联储**降息** / 非农弱 | 流动性宽松预期 → 风险偏好↑ | 成长、创新药、贵金属、港股科技 | — |
| 美股科技 / 费城半导体**大跌** | 情绪与估值映射 | — | A 股算力、半导体、CPO、光通信、存储 |
| 美股科技**大涨** / AI 资本开支超预期 | 情绪映射 + 产业链订单预期 | A 股算力、光模块、PCB、液冷 | — |
| **地缘冲突** / 油价大涨 | 避险 + 成本上升 | 军工、黄金、石油石化、油服 | 航空、高耗能制造 |
| 人民币**贬值** | 汇兑 + 资金流 | 出口链（家电/纺织/机械）、黄金 | 航空、进口依赖、外债高企业 |

### 内因（国内政策/景气）

| 消息 | A 股利好方向 | 备注 |
|---|---|---|
| 降准 / 降息 / 社融超预期 | 券商、银行、高估值成长 | L1 级，影响全局风险偏好 |
| 产业扶持政策落地 | 对应赛道（AI、低空、机器人、半导体设备等） | 看是否超预期、有无真实订单 |
| 大规模 IPO（如长鑫存储登陆科创板） | — | **利空**：分流存量科技资金，警惕流动性冲击 |
| 中报/年报披露窗口 | 业绩超预期个股 | 资金转向基本面验证，题材降温 |

> ⚠️ 映射表是**方向参考**，不是买入清单。同一消息在不同环境下强度不同，务必结合环境分级与预期差判断。

## 三、预期差三问（消息能不能用的过滤器）

对任何消息，动手前先问三句：

1. **超预期吗？** 市场已 price in 的消息无驱动力；只有**超预期 / 预期外**的消息才有真正的推动。
2. **已兑现吗？** 利好落地当天常见「**利好出尽是利空**」；利空充分预期后**钝化即机会**。
3. **时效多久？** 判断是**一日游**（脉冲，冲高即走）还是**趋势级**（可持续跟踪）。一日游消息只做情绪不做趋势。

> 三问任一为否定/不利，消息即降权或作废，不作为操作依据。

## 四、消息面纪律（守住不追热点的底线）

- **消息只用于调仓位、选方向，不作为唯一买入理由**；买入仍需个股三共振。
- **不追消息涨停板**：消息驱动的一字板/加速板是接盘区，不是入场点。
- **不信单一消息源**：重大消息需交叉验证（`search_news` + `search_notice` 双源）。
- **证伪即离场**：买入逻辑基于某消息，一旦消息被证伪或预期差消失，无条件走。
- **区分内外因**：不是所有海外消息都直接影响 A 股，先走传导路径判断，再定方向。

## 五、最新金融新闻取数协议（每次环境判断必执行）

> “最新”必须相对**本次决策时点**定义。先记录 `decision_time`（Asia/Shanghai），再发起检索；
> 不得把接口返回顺序直接当作时间顺序，也不得引用决策时点之后发布的内容。

### 5.1 默认观察窗口

| 场景 | `search_news` 默认窗口 | 重点 |
|---|---|---|
| 短线追涨 / 潜伏 / 异动 | 最近 3 个自然日 | 隔夜外盘、当日政策、突发事件、市场监管、热点产业 |
| 中线 / 账户周度复检 | 最近 30 个自然日 | 流动性、产业政策、供需价格、订单与景气拐点 |
| 长线 / 季度复检 | 最近 90 个自然日 | L1 宏观转向、制度变化、产业长期逻辑 |
| 个股体检 | 个股新闻最近 30 日；公告最近 90 日 | 公司事件、行业映射、传闻核验与治理风险 |

用户指定时间窗时优先使用用户口径。周末或节假日仍检索最新自然日新闻，但只作下一交易日的风险清单与方向储备。

### 5.2 最小查询集

按顺序执行，未知权益时控制为串行或每秒不超过 2 次：

1. **全市场查询**：宏观政策、流动性、海外市场、地缘风险、监管动态与 A 股重大风险事件。
2. **方向查询**：当前 Top 板块、候选行业或持仓产业链的最新政策、供需、价格、订单与风险。
3. **主体核验**：涉及具体公司且可能改变结论时，用 `search_notice` 和 `get_stock_events` 核验；
   L1 宏观数据用 `get_edb_data` 复核。

每次查询必须明确 `time_start`、`time_end` 和 `size`。单类默认取 5～10 条，避免用大量低相关转载淹没真正事件。

### 5.3 清洗、去重与时效校验

1. 从返回结果提取标题、内容片段、来源、URL、发布日期/时间和查询工具；缺少发布日期时标记“时间不明”。
2. 过滤发布时间晚于 `decision_time` 的内容；接口只给日期时，不得声称获得了精确到小时的盘中先后关系。
3. 按“规范化标题 + 来源事件 + 发布日期”聚类；同一政策、同一发布会或同一传闻的转载只算**一个事件**。
4. 聚类内保留最接近一手来源、发布时间最清晰、内容最完整的记录，其余仅作为交叉验证。
5. 按发布时间倒序输出，再按 L1→L4、事件置信度排序；语义检索命中顺序不等于新闻发布时间顺序。
6. 时效标签统一为：`≤1日=最新`、`2～3日=近期`、`>3日=背景`。中长线可保留背景事件，但不得称为最新催化。

## 六、新闻情绪归纳（按事件，不按文章数量）

每个去重事件统一标注：

| 字段 | 可选值 / 说明 |
|---|---|
| `level` | L1 / L2 / L3 / L4 |
| `direction` | 利多 / 利空 / 中性 / 分歧 |
| `scope` | 全市场 / 行业板块 / 个股 |
| `freshness` | 最新 / 近期 / 背景 / 时间不明 |
| `expectation` | 超预期 / 符合预期 / 已定价 / 无法判断 |
| `market_confirmation` | 引用外围模块和A股盘面：已确认 / 未确认 / 反向运行 / 数据缺失；不得凭新闻标题填写 |
| `confidence` | 高：权威或公告双源；中：多家可靠媒体一致；低：单一媒体、观点稿或传闻 |

**聚合纪律**：

- 同一事件的 20 篇转载不能放大为 20 份情绪；只形成一个事件及一个方向判断。
- 观点稿、券商预测和标题情绪不能代替事实；事实与观点必须分栏。
- 利多与利空并存时输出“分歧”，不得强行合成为单一看多/看空结论。
- 新闻情绪只输出 `偏积极 / 中性 / 偏谨慎 / 高风险` 四档，不生成伪精确的 0～100 分。
- 最终档位必须结合指数、涨跌停、资金与板块表现；新闻与盘面背离时，以“分歧待确认”处理。

## 七、如何修正环境分级（接口约定，写给 Agent）

```text
1. 先按《大盘环境模块》取四维数据，计算 0-100 原始环境分与初始分级。
2. 读取《外围市场影响模块》的 O0—O4、置信度、`overlay_id`、禁区和已执行动作。
3. 按第五节拉取截至 decision_time 的最新金融新闻，并按第六节去重归纳。
4. 再做一次最终定性修正（不改环境分值）：

   IF 重大系统性利空尚未被外围价格层执行，且达到高置信度
   THEN 最终分级下调一档，并设置受冲击板块禁区，`adjustment_owner=news_event`。

   IF 同一冲击已由外围价格层执行 O3/O4
   THEN 关联同一 `overlay_id`，只补充原因、映射、置信度或有效期，`adjustment_owner=merged`。

   IF 出现高置信度 L1/L2 超预期利好，且盘面尚未过度兑现
   THEN 不上调环境分级，只将对应方向纳入主线加权参考。

   IF 利好已充分兑现，或新闻与价格出现“利好不涨”背离
   THEN 不加分，对相关板块启动见顶警示并收紧止盈。

   IF 新闻方向冲突、来源置信度低或价格未确认
   THEN 标记“分歧待确认”，不得改变硬门槛。

5. 将唯一的最终分级、禁区、观察方向、`adjustment_owner` 和证据链交回对应策略。
6. 周末/节假日消息仅作方向储备，不直接触发实时操作。
```

## 十、失败处理与能力边界

- `search_news` 返回的是**语义相关片段**，不是全量新闻流，也不保证天然按时间倒序。
- 无结果只代表本次检索未命中，不能推断“市场无消息”“公司无利空”或“情绪中性”。
- 查询失败、权益不足、时间字段缺失或结果截断时，必须输出缺失项；涉及硬否决时按 fail-closed 处理。
- 重大事件至少需要“新闻 + 公告/结构化事件/宏观数据/权威来源”中的两类证据；单一传闻不能改变环境分级。
- 不缓存未知时点的“最新结论”。若调用方使用缓存，盘中新闻 TTL 建议不超过 5 分钟，一般新闻不超过 15 分钟，
  公告不超过 60 分钟，并在报告中标注缓存时间与是否过期。
- 不把新闻数量、情感词数量或媒体标题直接映射成仓位；仓位仍受环境分、策略硬闸与账户风险预算约束。

## 十一、流程示例（执行时必须重新真实取数）

- **假设事件**：海外大型科技公司削减 AI 资本开支，引发科技链估值担忧。
- **价格层**：若已完成的美股、港股或韩国相关资产达到 O3/O4 条件，由《外围市场影响模块》生成唯一风险动作。
- **新闻层**：检索权威来源解释原因、判断是否超预期，并关联同一 `overlay_id`；不得再次下调。
- **A股层**：相关板块只有在 A 股盘面也确认时才延长禁区；避险方向仅列观察，不盘前押注。
