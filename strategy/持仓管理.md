# 持仓管理（台账规范 · 维护流程）

> 生成日期：2026-07-02
> 本文定义**持仓台账的字段规范与维护流程**，是《账户分析策略》《卖出决策策略》《资金与风险管理策略》能落地执行的**数据载体**。
> 台账数据存放在 `account/{账户名}.csv`，本文规定它应该有哪些列、怎么维护。
>
> **⚠️ 重要声明：本文为持仓记录管理规范，不构成任何投资建议。请独立决策、盈亏自负。股市有风险，投资需谨慎。**

---

## 〇、为什么需要台账

没有台账，下面这些都无法执行：
- **固定风险法仓位** → 需要账户总额
- **总仓位/现金比例** → 需要账户总额 + 各仓市值
- **跨线集中度约束** → 需要每票的行业/归属线
- **定期复检** → 需要入场价、止损位、下次复检节点
- **复盘校准** → 需要买入逻辑、盈亏结果

**台账是整个系统从"分析"走向"管理"的地基。**

## 一、台账字段规范（`account/{名}.csv`）

### 必填列（现有）
| 列名 | 含义 | 示例 |
|------|------|------|
| 股票名称 | 证券简称 | 紫光国微 |
| 股票代码 | 6 位代码（**务必与名称核对，防写错**） | 002049 |
| 成本线 | 每股持仓成本 | 88.75 |
| 仓位（股） | 持有股数 | 400 |

### 建议补充列（让管理闭环）
| 列名 | 含义 | 用途 |
|------|------|------|
| 策略归属 | 短线/中线/长线 | 决定用哪条线的纪律诊断（止损宽度/复检频率） |
| 建仓日期 | 首次买入日 | 估算持有周期、加仓间隔 |
| 止损位 | 已设定的止损价 | 复检时判断是否触损 |
| 买入逻辑 | 一句话理由 | 复检时验证逻辑是否兑现、复盘归因 |
| 已加仓次数 | 长线越跌越买计数 | 配合分批纪律，防超上限摊平 |

### 账户级元信息（单独维护）
> CSV 不便存账户级信息，建议在 `account/{名}_meta.txt` 或台账顶部注释记录：
- **账户总额**（含现金）—— 固定风险法、总仓位的分母，**最关键**
- **现金余额**
- **本月起始净值**（算月度回撤用）

## 二、台账维护流程

| 时机 | 动作 |
|------|------|
| **每次买入后** | 追加/更新该票的 成本线、股数、建仓日期、止损位、买入逻辑、策略归属 |
| **每次加仓后** | 更新成本线（加权）、股数、已加仓次数 |
| **每次卖出后** | 减少股数或删除该行，并把该笔记入复盘日志（见各线复盘模板） |
| **每次分红送转** | 调整成本线与股数 |
| **定期复检时** | 按归属线频率（短线次日/中线每周/长线季报后）核对止损位、复检结论 |

---

## 三、数据质量铁律

1. **代码必须与名称核对**——写错代码会导致自动分析拉错票（如把"紫光国微"写成 002048 实为宁波华翔）。录入后用 MCP 核对一次名称。
2. **成本线含费用**——尽量用含佣金/税的实际成本，误差影响浮盈亏判断。
3. **账户总额必填**——否则总仓位、单笔风险、集中度占比全部无法算。
4. **卖出即归档**——卖出后把交易记入复盘，不要直接删了事，复盘是校准的数据来源。

## 四、台账模板（可复制）

**`account/{名}.csv`（增强版表头）**
```csv
股票名称,股票代码,成本线,仓位（股）,策略归属,建仓日期,止损位,买入逻辑,已加仓次数
紫光国微,002049,88.75,400,中线,2026-06-30,83.0,芯片景气+资金持续流入,0
```

**`account/{名}_meta.txt`（账户元信息）**
```
账户总额（含现金）：1000000
现金余额：650000
本月起始净值：980000
```

## 五、与其他策略的衔接

| 需求 | 台账提供的字段 | 使用策略 |
|------|----------------|----------|
| 逐票诊断 | 全部持仓行 | 账户分析策略 |
| 仓位计算 | 账户总额 + 止损位 | 资金与风险管理策略 |
| 集中度检查 | 各票市值 + 行业（MCP 补） | 资金与风险管理策略 |
| 卖出决策 | 成本线 + 止损位 + 买入逻辑 | 卖出决策策略 |
| 复盘校准 | 买入逻辑 + 盈亏结果 | 各线复盘模板 |
