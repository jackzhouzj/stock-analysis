# 持仓管理（台账规范 · 维护流程）

> 生成日期：2026-07-02
> 本文定义**持仓台账的字段规范与维护流程**，是《账户分析策略》《卖出决策策略》《资金与风险管理策略》能落地执行的**数据载体**。
> 台账数据存放在 `account/{账户名}.csv`，本文规定它应该有哪些列、怎么维护。
>
> **⚠️ 重要声明：本文为持仓记录管理规范，不构成任何投资建议。请独立决策、盈亏自负。股市有风险，投资需谨慎。**

---

## 〇、为什么需要台账

没有台账，下面这些都无法执行：
- **压力风险法仓位** → 需要账户净资产、压力退出价、流动性与组合剩余风险额度
- **总仓位/现金比例** → 需要账户总额 + 各仓市值
- **跨线集中度约束** → 需要每票的行业/归属线
- **定期复检** → 需要入场价、止损位、下次复检节点
- **复盘校准** → 需要买入逻辑、盈亏结果

**台账是整个系统从"分析"走向"管理"的地基。**

## 一、台账字段规范（`account/{名}.csv`）

### 必填列
| 列名 | 含义 | 示例 |
|------|------|------|
| 股票名称 | 证券简称 | 紫光国微 |
| 股票代码 | 6 位代码（**务必与名称核对，防写错**） | 002049 |
| 成本线 | 每股持仓成本 | 88.75 |
| 仓位（股） | 持有股数 | 400 |
| 策略归属 | 建仓时冻结的短线/中线/长线 | 中线 |
| 买入逻辑 | 建仓时记录的可证伪理由 | 芯片景气+资金持续流入 |

### 建议补充列（让订单、风险与复盘闭环）
| 列名 | 含义 | 用途 |
|------|------|------|
| 策略归属 | 建仓时冻结的短线/中线/长线 | 决定纪律；禁止因被套按持仓时长改线 |
| 建仓日期 | 首次买入日 | 估算持有周期、加仓间隔 |
| 止损位 | 已设定的止损价 | 复检时判断是否触损 |
| 买入逻辑 | 一句话可证伪理由 | 复检时验证逻辑是否兑现、复盘归因 |
| 已加仓次数 | 已执行的加仓次数 | 防超上限摊平 |
| 交易批次ID | 每次买入 lot 的唯一标识 | 区分成本、可卖日期与退出归因 |
| 买入日期 | 该 lot 的成交日期 | 计算 T+1 可卖日期 |
| 可卖日期 | 该 lot 最早合法卖出日 | 防止回测/操作假设当日可卖 |
| 可卖数量 | 当前可卖股数 | 区分总持仓与冻结数量 |
| 订单状态 | 待报/已报/部分成交/已成交/已撤/拒单 | 不把委托等同于成交 |
| 未成交数量 | 剩余未成交股数 | 持续占用风险额度并安排后续退出 |
| 累计费用 | 佣金、印花税、过户费等 | 计算真实成本与净收益 |
| 交易计划版本 | 建仓时采用的策略/阈值版本 | 保证复盘可重现 |

### 账户级元信息（必须独立维护）
> 建议使用 `account/{名}_meta.csv` 或可机器读取的结构化文件，至少每日记录：
- 日期、账户净资产、现金余额、持仓市值、可用资金。
- 累计入金、累计出金、当日净入出金。
- 当日已实现盈亏、未实现盈亏、交易费用。
- 月初净值、历史高水位、当日/本月/历史最大回撤。
- 组合开放风险、未成交买卖订单风险和当前熔断状态。

## 二、台账维护流程

| 时机 | 动作 |
|------|------|
| **每次买入/加仓成交后** | 每个成交 lot 新增独立行，更新成本、数量、买入/可卖日期、费用、策略归属和计划版本；策略归属不得因被套变更 |
| **委托状态变化时** | 更新订单状态、已成交/未成交数量；未成交止损继续占用组合开放风险并冻结加仓 |
| **每次卖出后** | 按实际成交 lot 减少数量，记录成交价与费用；完全卖出后归档，不直接删除审计记录 |
| **每次分红送转** | 按公司行动调整批次成本、股数与总回报链，保留调整前后记录 |
| **每日收盘后** | 写入账户净资产、现金、入出金、已实现/未实现盈亏、费用、高水位、回撤和开放风险快照 |
| **定期复检时** | 按冻结归属线频率核对止损位、复检结论、熔断状态和恢复条件 |

## 三、数据质量铁律

1. **代码必须与名称核对**——名称与代码不匹配会导致自动分析拉错证券。录入后必须用 MCP 核对证券代码、简称与市场。
2. **成本线含费用**——尽量用含佣金/税的实际成本，误差影响浮盈亏判断。
3. **账户净资产必填**——否则总仓位、单笔风险、集中度和组合开放风险均不可执行；不得用持仓市值替代正式分母。
4. **订单与净值必须可追溯**——委托不等于成交；每日快照必须能用前一日净值、净入出金、当日盈亏和费用勾稽，无法勾稽时禁止计算熔断恢复。
5. **卖出即归档**——卖出后保留 lot、订单与交易记录，复盘是校准的数据来源。

## 五、与其他策略的衔接

| 需求 | 台账提供的字段 | 使用策略 |
|------|----------------|----------|
| 逐票诊断 | 全部持仓行 | 账户分析策略 |
| 仓位计算 | 账户总额 + 止损位 | 资金与风险管理策略 |
| 集中度检查 | 各票市值 + 行业（MCP 补） | 资金与风险管理策略 |
| 卖出决策 | 成本线 + 止损位 + 买入逻辑 | 卖出决策策略 |
| 复盘校准 | 买入逻辑 + 盈亏结果 | 各线复盘模板 |
