# 复盘与进化策略（统一复盘中枢 · 数据驱动的策略迭代）

> 生成日期：2026-07-02
> 三条线各有复盘模板，但缺一个**统一的复盘中枢**把它们收口。本策略统一交易日志口径、汇总周期、策略有效性评估，并把校准结果收敛到 `阈值与版本记录.md`，形成 **数据 → 复盘 → 校准 → 版本记录** 的完整闭环。
> **复盘是策略从"抄来的经验"进化成"自己的系统"的唯一路径。没有复盘数据，所有胜率/阈值都只是感觉。**
>
> **⚠️ 重要声明：本文为复盘方法论框架，不构成任何投资建议。请独立决策、盈亏自负。股市有风险，投资需谨慎。**

---

## 〇、核心原则

- **没有记录就没有复盘，没有复盘就没有进化**：每笔交易必留痕。
- **复盘归因到"环节"，不归因到"运气"**：亏了要能说清是环境、板块、个股、还是纪律哪一环出问题。
- **阈值不是拍脑袋定的，是复盘校准出来的**：经验默认值只是起点，用实测数据逐步个人化。

---

## 一、统一交易日志（每笔必填）

> 各线复盘模板字段略有差异，本表是**跨线统一的最小字段集**，落地到交易日志（建议 `account/{名}_journal.md` 或表格）。

| 字段 | 说明 |
|------|------|
| 日期 / 标的 | 交易日 + 股票名称(代码) |
| 归属线 | 短线 / 中线 / 长线 |
| 方向 | 建仓 / 加仓 / 减仓 / 清仓 |
| 入场逻辑 | 触发的闸门/形态/基本面理由 |
| 打分 / 档位 | 严格筛选打分卡结果 |
| 盈亏比（入场前算的） | 上行空间 ÷ 下行空间 |
| 仓位 | 占账户 % |
| 结果 | 盈利/亏损 +X% / -X%，持有周期 |
| 卖出原因 | 止损/止盈/证伪/换仓/治理雷 |
| **对错归因** | 环境 / 板块 / 个股 / 纪律，哪个环节 |
| 教训 | 下次同样情况怎么做 |

---

## 二、汇总周期（不同线不同节奏）

| 周期 | 适用 | 汇总内容 |
|------|------|----------|
| **月度** | 短线 | 交易笔数、胜率、盈亏比、最大回撤、摩擦成本、月度教训 |
| **季度** | 中线 | 胜率、平均持有周期、趋势判断正确率、景气验证兑现率 |
| **年度** | 长线 | 组合收益 vs 沪深300、买入逻辑兑现率、估值判断准确率 |

> 三条线的详细模板保留在各自严格筛选策略的"复盘记录模板"里；本中枢负责**汇总口径统一**和**跨线对比**。

## 三、策略有效性评估

> 定期回答：我的策略到底赚不赚钱、哪条线有效、问题出在哪。

| 评估维度 | 关键指标 | 判断 |
|---|---|---|
| 整体盈利能力 | 期望值 = 胜率×平均盈利 − 败率×平均亏损 | >0 才是正期望系统 |
| 哪条线有效 | 分线统计胜率/盈亏比/贡献 | 长期负期望的线 → 降频或暂停 |
| 纪律执行度 | 实际止损/止盈 vs 计划的偏差率 | 偏差大 = 纪律问题，非策略问题 |
| 摩擦成本占比 | 年交易成本 / 本金 | 过高 → 降低交易频率 |
| 最大回撤 | 账户历史最大回撤 | 触及保命线（见资金管理）→ 系统性检讨 |

> **区分"策略无效"和"没执行策略"**：很多亏损是没守纪律，不是策略错了——归因要诚实。

## 四、校准闭环（把实测值回填到系统）

```
数据（交易日志）
  → 复盘（月/季/年汇总 + 有效性评估）
  → 校准（用实测胜率/兑现率修正阈值与权重）
  → 记录（写入 阈值与版本记录.md）
  → 下一周期按新阈值执行
```

各线校准要点（详见各严格筛选策略的"校准"小节）：
- **短线**：实测形态胜率回填打分卡权重、重估形态优先级、校准"启动涨幅"等硬阈值。
- **中线**：趋势判断正确率、景气验证兑现率偏低 → 收紧对应闸门。
- **长线**：买入逻辑兑现率、估值判断准确率偏低 → 收紧质量/成长/估值闸。

> **所有阈值调整必须写入 `阈值与版本记录.md`**，标注调整原因与数据依据，禁止随手改、无痕改。

---

## 五、复盘纪律

1. **交易当下就记录入场逻辑**（事后补记会美化/失真）。
2. **亏损单更要复盘**——利润会说谎，亏损才见真章。
3. **归因到环节，不甩锅给市场**。
4. **阈值调整留痕**，可回溯（见版本记录）。
5. **定期（至少每季）做一次策略有效性评估**，决定加码/降频/暂停哪条线。

## 一句话总览

> **每笔留痕、定期汇总、诚实归因、数据校准、留痕迭代**——复盘把经验值换成实测值，是策略唯一的复利来源。
