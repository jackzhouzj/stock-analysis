## 字段陷阱

### kaipan 强度分单位

- `getPlateRotatData` `from=kaipan` 返回的数值是**纯整数**: `15199`
- 不带 `%`,parsers 用 `value_type='score'` 标注
- 单位是开盘啦内部"强度分" — 上榜次数 + 涨速 + 龙头数等多因子综合
- **数字本身没有外部可解读语义,只用于排序**

### kaipan 源仅认 80x / 803x 前缀板块

- 试图传 88x 板块代码到 `from=kaipan` 路径的接口会返回空数据
- 80x = 板块体系核心 (`801807` 算力 / `801660` 通信)
- 803x = 概念衍生 (`803023` AI 应用)
- 二者都属于开盘啦,可以混用

## 强度分 vs 涨幅 % 的解读哲学

- 涨幅 % = "**今天谁在爆发**" (瞬时资金集中度)
- 强度分 = "**这条赛道还在不在跑**" (持续性 + 涨速 + 龙头数综合)
- 同一板块同一天两个数字**不同步**很正常: 强度分高但当日不涨 = 蓄势中
- 两边都上榜 = 真主线;只在 ths 上榜 = 偶发热点;只在 kaipan 上榜 = 老热点退潮中
